A Programação Orientada a Objeto (POO) é separada em quatro comportamentos:

Herança : Onde um objeto FILHO herda de um outro objeto PAI, assim como o PAI pode ser herdado de um outro objeto também. São boas práticas para se ter um Reuso de Códigos.

Polimorfismo : Após um objeto herdado pode ser alterado características desse objeto.

Encapsulamento : Onde pode esconder objetos para que o usuário possa ver somente o que é permitido pelo programador, isso o torna um objeto privado. O usuário só pode ver o que é denominado como publico pelo desenvolvedor, isso é uma boa prática de segurança.

Abstração : Representa um objeto de uma forma abstrata, ou seja, tudo o que se vê na vida real a olho nú passa-se para a linguagem do computador.
